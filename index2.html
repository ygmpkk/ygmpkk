Pyton del函数

Wed 10 April 2013
By timothy

In python.

tags: python

前几天看到有人提议说， 开一个程序员主题餐厅，LOL， 这个想法好，C程序员必须光棍进来， C++/Delphi可有可无，Java/C#必须带对象来， 走的时候，c++的必须独立离开，java的可以一起走。

Python/PHP/Javascript也可有可无，Ruby也必须带对象来； Erlang必须一边说话走来； Lisp必须嵌套着来； 哈哈哈

还有垃圾回收？

C/C++/Delphi必须自己把垃圾带走 Lisp/Java/Erlang/Python/Ruby可以不带走垃圾，清洁工定期回收；

Python如何清理垃圾，使用del函数； 但是python的del函数，和c++的delete不一样；

在python的交互模式下运行，>>> 是提示符

>>> objA = object()
>>> objB = objA
>>> dictA = {'O1': objA,'O2': objB}
>>> dictB = dictA['O1']

看objA和objB的内存地址

>>> objA
<object object at 0x10270a0a0>
>>> objB
<object object at 0x10270a0a0>

内存地址都是一样的

>>> del objB
>>> objA
<object object at 0x10270a0a0>
>>> objB
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
NameError: name 'objB' is not defined

看到del仅仅删除objB的引用，而不是删除objA的对象，object()依然存在与内存上；

在dict的

>>> dictA
{'O2': <object object at 0x10270a0a0>, 'O1': <object object at 0x10270a0a0>}
>>> del dictB
>>> dictB
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
NameError: name 'dictB' is not defined
>>> dictA['O1']
<object object at 0x10270a0a0>
>>> dictA['O2']
<object object at 0x10270a0a0>

dict会把object()的对象放在value上，而不是引用objB；

仅仅删除objB也就不能删除dictA里面的值

>>> del dictA['O1']

这样就能够把dict的值删掉

see aslo: http://docs.python.org/2/tutorial/datastructures.html#the-del-statement/
